# GEE: источники и параметры подготовки слоев

Этот файл документирует, как формируются GeoTIFF-слои в папке `data/` через скрипты из `gee/`.

## 1) Коллекции и даты

| Слой | Скрипт | GEE-коллекция | Период |
|---|---|---|---|
| Температура поверхности (LST) | `landsat_lst.js` | `LANDSAT/LC08/C02/T1_L2` + `LANDSAT/LC09/C02/T1_L2` | `2025-06-01` — `2025-08-31` |
| Диоксид азота (NO2) | `tropomi_no2.js` | `COPERNICUS/S5P/OFFL/L3_NO2` | `2023-01-01` — `2023-12-31` |
| Ночное световое загрязнение | `viirs_nightlight.js` | `NOAA/VIIRS/DNB/MONTHLY_V1/VCMSLCFG` | `2023-01-01` — `2023-12-31` |
| Акустическая модель | `acoustic_noise.js` | `WorldPop/GP/100m/pop`, `NOAA/VIIRS/DNB/MONTHLY_V1/VCMSLCFG`, `USGS/SRTMGL1_003`, `ESA/WorldCover/v200/2021` | WorldPop: `2020`, VIIRS: `2023`, WorldCover: `2021`, SRTM: статический |

## 2) QA-фильтры

| Слой | QA-фильтр |
|---|---|
| LST (Landsat) | Используется `QA_PIXEL`: маскируются облака и тени через биты `3` и `4` (`bitwiseAnd(...).eq(0)`) |
| NO2 (TROPOMI OFFL) | В текущем скрипте явный `qa_value`-фильтр не задан |
| Свет (VIIRS) | В текущем скрипте дополнительный QA-фильтр не задан |
| Шум (модель) | QA не применим напрямую, это расчетный слой из нескольких источников |

## 3) Агрегирование и постобработка

| Слой | Базовая агрегация | Сглаживание / постобработка |
|---|---|---|
| LST | `median()` по летней коллекции | `focal_mean(radius: 200m)` |
| NO2 | `median()` по годовой коллекции после конвертации в микромоли | `focal_mean(radius: 5000m)` |
| Свет | `median()` по `avg_rad` | `focal_mean(radius: 500m)` |
| Шум | Аддитивная модель: базовый фон + вклад населения + трафик + застройка | `focal_mean(radius: 800m)` + вычитание демпфирования по уклону и лесу |

## 4) Разрешение (resolution)

- Во всех экспорт-операциях используется `scale: 250` (см. `Export.image.toDrive`).
- По метаданным текущих `data/*.tif` фактический шаг сетки: около `0.002245788°/px`, что соответствует примерно `~250 м/px` в широтах Крыма.

## 5) CRS

- CRS экспортированных слоев в `data/*.tif`: **EPSG:4326 (WGS 84)**.
- Это подтверждается GeoTIFF GeoKeys (`GeographicTypeGeoKey = 4326`).

## 6) Примечание для воспроизводимости

Чтобы уменьшить вариативность, рекомендуется:
1. Зафиксировать одинаковые диапазоны дат в GEE (как в таблице выше).
2. Не менять `scale`, `region`, `median()` и радиусы `focal_mean` между пересборками.
3. При необходимости добавить явные QA-фильтры для `NO2` и `VIIRS` как отдельный commit, чтобы не смешивать с финальной демо-веткой.
